### 1 [00]
Use a literal constant, load it to A, then store the whole word to X.

```
        LDA   =+1000=     * A ← 1000 (literal)
        STA   X           * X ← 1000 (stores full word, field 0:5)
...
X       CON   0           * (somewhere: reserve the cell for X)
```

`X EQU 1000` only defines a symbol; it doesn’t write memory. `LDA =1000=` makes the value 1000 available at run time, and `STA X` sets the memory cell named `X` to 1000.

### 2 [10]

Because the word labeled `EXIT` (which contains `JMP *`) is self-modified at entry to the subroutine.

Line 3 does `STJ EXIT 1`. In MIX, `STJ …,1` stores the contents of `rJ` (the caller’s return address) into the address field of the memory word at `EXIT`. Thus the instruction at `EXIT` is changed from

```
JMP *
```

(where `*` initially means "this location") to

```
JMP <return address>
```

When line 12 executes, it jumps to that stored return address, not to itself, so there is no infinite loop.

### 3 [23]

It’s a selection-sort wrapper around subroutine **M (MAXIMUM)**.

1. `IN X+1(0)` / `JBUS *(0)` reads 100 words into `X[1]..X[100]` and waits.
2. `ENT1 100` sets `n ← 100`.
3. Loop (`1H … J1P 1B`):

   * `JMP MAXIMUM` calls Program M, which returns the maximum of `X[1..n]` in `A` and its index in `rI2 = j`.
   * `LDX X,1` loads `X[n]` into `X`.
   * `STA X,1` stores the max at `X[n]`.
   * `STX X,2` stores the old `X[n]` at `X[j]` (swap).
   * `DEC1 1` decrements `n`; repeat while `n > 0`.
4. `OUT X+1(1)` outputs the block; `HLT` stops.

Effect: it sorts the 100 input numbers in nondecreasing (ascending) order, placing the largest remaining element at the end on each pass, then prints the sorted list.

### 5 [11]
Because Program P does a *lot* of work between two output operations and it double-buffers the line being printed.

`OUT 0,4(PRINTER)` starts the printer asynchronously and the program immediately switches `rI4` to the other buffer (`LD4 24,4`). ### 6 [HM20]

(a) If $n$ is composite, write $n=ab$ with $1<a\le b<n$.
If both $a>\sqrt n$ and $b>\sqrt n$, then $ab>\sqrt n\cdot\sqrt n=n$, a contradiction.
Hence at least one factor is $\le\sqrt n$. Let $d=\min(a,b)$; then $1<d\le\sqrt n$.

(b) In Program **P**, step **P6** divides $N$ by the current prime $p=\text{PRIME}[k]$:
$N = pQ + R$ with $0\le R<p$. MIX puts $Q$ in $rA$ and $R$ in $rX$.
If $R=0$ the program declares “not prime”.

Step **P7** (the code `CMPA PRIME,3; INC3 1; JG 6B`) compares that quotient $Q$ with the **next** prime $p'=\text{PRIME}[k+1]$ and loops back to **P6** exactly when $p' \le Q$.

Why does stopping when $p' > Q$ prove primality?
If $p\le\sqrt N$ then $\displaystyle Q=\left\lfloor \frac{N}{p}\right\rfloor \ge \sqrt N$.
Therefore any prime divisor $\le\sqrt N$ is $\le Q$. Program **P** tries primes in increasing order; as long as there could still be a prime $\le\sqrt N$ left, we have $p' \le Q$ and the loop continues. When the test first finds $p' > Q$, every remaining prime exceeds $Q \ge \sqrt N$. Since we have already checked all smaller primes (and none divided $N$), $N$ has no divisor $\le\sqrt N$. By part (a), $N$ must be prime.

### 7 [10]

(a) In MIXAL a digit with "B" means "the nearest **previous** line whose location is that digit followed by H."
So the `4B` in line 34 means "jump to the most recent `4H` before this line," which here is line 29 (address 3020). Thus `J5P 4B` jumps back to 3020 when rI5 > 0.

(b) If we relabel line 15 as `2H` and change the target of line 20 to `2B`:

- Line 20 will still jump to line 15 (because `2B` now refers to the nearest previous `2H`, which would be the newly labeled line 15). So that branch is unchanged.
- But line 24 already contains `JMP 2B`. After the relabel, its target changes from the old `2H` at line 12 (step P2: store the prime and increment $J$) to the new `2H` at line 15 (step P4: advance $N$). That means whenever $N$ is prime the program would skip storing it and skip incrementing $J$. Consequently $J$ never reaches 500, so the "500 found?" test never triggers and the printing phase never runs—effectively breaking the program (it will keep searching forever).

So: (a) "previous 4H" (line 29). (b) The change is not harmless; it makes line 24 jump to the wrong place and the program fails to record primes or terminate.

### 8 [24]

Here's the original "mystery program" rewritten with **inline MIXAL comments** so you can see exactly what every line does:

```
* MYSTERY PROGRAM (annotated)

BUF   ORIG **+3000          * Define BUF as an address 3000 words ahead of here.
                            * Code that follows runs before BUF; BUF.. is our data area.

1H    ENT1 1                * I1 ← 1           (run-length to write)
      ENT2 0                * I2 ← 0           (write pointer into BUF)
      LDX  4F               * X  ← word at 4H  (ALF AAAAA) → X="AAAAA"

2H    ENT3 0,1              * I3 ← 0+I1        (copy I1 into I3: loop counter)

3H    STZ  BUF,2            * BUF[I2] ← +0     (store a blank word)
      INC2 1                * I2++             (advance pointer)
      DEC3 1                * I3--             (one blank done)
      J3P  3B               * if I3>0, repeat from 3H (write exactly I1 blanks)

      STX  BUF,2            * BUF[I2] ← X      ('AAAAA' after the blanks)
      INC2 1                * I2++             (advance after the 'A' word)
      INC1 1                * I1++             (next time, one more blank)
      CMP1 =75=             * compare I1 with 75
      JL   2B               * if I1<75, go back to 2H and build next block

      ENN2 2400             * I2 ← −2400       (set up to print from BUF upward)
      OUT  BUF+2400,2(18)   * print one line (24 words) starting at BUF+2400+I2;
                            * device 18 = line printer. With I2<0 this starts at BUF.
      INC2 24               * I2 += 24         (next line starts 24 words later)
      J2N  *-2              * while I2<0, jump back two lines to OUT (prints 100 lines)
      HLT                   * done

4H    ALF  AAAAA            * AAAAA literal used above (loaded into X)
      END  1B               * entry is at label 1H
```

**What it does**

Phase 1 (building the buffer). Starting at `BUF`, the program writes a pattern:
for `k = 1, 2, …, 74`, it stores **k blank words** (each is `+0`, which prints as five spaces) followed by the word **`AAAAA`**. After the last iteration (`k=74`) it stops. The total words written are
$\sum_{k=1}^{74}(k+1) = 2849$.
The positions of the `AAAAA` words within the buffer are

$$
p_m=\sum_{k=1}^{m}(k+1)=\frac{m(m+3)}{2}\quad(m=1,2,\dots,74)
$$

i.e., at word numbers $2,5,9,14,20,27,\dots$ with gaps growing by $3,4,5,\ldots$.

Phase 2 (printing). It then prints the **first 2400 words** of this buffer as **100 lines** of the line printer (24 words = 120 characters per line), starting at `BUF`, then `BUF+24`, `BUF+48`, …, `BUF+2376`. Because blank words render as spaces and the nonblank words render as `AAAAA`, the page shows a sequence of `AAAAA` blocks separated by ever-increasing runs of spaces—a triangular/“quadratically spaced” pattern of A’s drifting across the printed lines.

### 9 [25]
Here’s a clean, table-driven MIXAL solution that validates a word in `INST`.
It jumps to `GOOD` iff the instruction has a valid C-field (opcode), ±AA field (address) when required, I-field (index), and F-field; otherwise it jumps to `BAD`.

```mixal
* EX 1.3.2–9 — Verify that INST is a legal MIX instruction.
* If legal, jump to GOOD; otherwise jump to BAD.

            ORIG 3000

* ---------- Small constants and helpers --------------------------------------

B       EQU  1(4:4)          * MIX byte size (b = 64) via the canonical trick
BMAX    EQU  B-1             * for MOVE: F ∈ [0..b-1]
UMAX    EQU  20              * highest I/O unit number in the MIX spec

* VALID packs several convenient bounds into one word:
* (3:3)=6  → upper bound for I
* address=3999 → largest legal memory address
* (4:4)=6  → “floating” F used by arithmetic ops (ADD/SUB/MUL/DIV)
VALID   CMPX 3999,6(6)

INST    CON  0               * (caller puts candidate instruction here)

* ---------- Common front-end checks + table dispatch --------------------------

BEGIN   LDA   INST                   * A ← INST (full word)
        CMPA  VALID(3:3)             * compare I with 6
        JG    BAD                    * I > 6 → illegal

        LD1   INST(5:5)              * I1 ← opcode (C)
        DEC1  64
        J1NN  BAD                    * C ≥ 64 → illegal (not an instruction)

        CMPA  TABLE+64,1(4:4)        * compare F with per-opcode Fmax from table
        JG    BAD                    * F > Fmax → illegal

        LD1   TABLE+64,1(1:2)        * I1 ← address of per-opcode checker
        JMP   0,1                    * jump there

* ---------- Per-opcode checker routines --------------------------------------

* FLOAT — arithmetic ops: either a valid partial field or F==6 (“floating”)
FLOAT   CMPA  VALID(4:4)             * F == 6 ?
        JE    MEMORY                 * if yes, only the address rule remains
        JMP   FIELD                  * else check partial field as usual

* FIELD — validate F as 8L+R with 0 ≤ L ≤ R ≤ 5
* Trick: let F = 8L+R, divide by 9 →  A = ⌊F/9⌋ and X = F mod 9.
* - If R ≥ L:  ⌊F/9⌋=L, rem=R-L → A+rem = R → require R ≤ 5.
* - If R < L:  ⌊F/9⌋=L−1, rem=R−L+9 → A+rem = R+8 ≥ 8 → reject.
FIELD   ENTA  0
        LDX   INST(4:4)              * X ← F
        DIV   =9=
        STX   *+1(0:2)               * self-modify the next INCA with rem
        INCA  0                      * A ← ⌊F/9⌋ + (F mod 9)
        DECA  5
        JAP   BAD                    * if >5 → illegal
        JMP   MEMORY                 * finally validate the address if needed

* MEMORY — when I=0, address must be a valid memory/unit number
MEMORY  LDX   INST(3:3)              * X ← I
        JXNZ  GOOD                   * if I ≠ 0, indexing OK → done
        LDX   INST(0:2)              * X ← ±AA (or device/unit for I/O)
        JXN   BAD                    * negative address/unit not allowed
        DECX  3999
        JXNP  GOOD                   * 0..3999 is OK (table narrows bound for I/O)
        JMP   BAD

GOOD    JMP   GOOD                   * (return address patched by caller)
BAD     JMP   BAD

* ---------- Opcode table: Fmax in (4:4), checker address in (1:2) -------------
* The pattern is what matters; you can fill remaining opcodes analogously.
* N.B.: For opcodes that do not use ±AA, the checker is just GOOD.
*       For I/O and MOVE, the checker is MEMORY with the appropriate bound.

TABLE   NOP    GOOD(BMAX)            * C=0  : any F up to B−1 is harmless
        ADD    FLOAT(5:5)            * C=1  : arithmetic allow F=6 (‘floating’)
        SUB    FLOAT(5:5)            * C=2
        MUL    FLOAT(5:5)            * C=3
        DIV    FLOAT(5:5)            * C=4
        HLT    GOOD                  * C=5  : F ignored; no address use
        SRC    GOOD                  * C=6  : shifts/rotates; address is count
        MOVE   MEMORY(BMAX)          * C=7  : any F in [0..B−1]; addr must be valid
        LDA    FIELD(5:5)            * C=8  : partial fields only
* ...   (fill remaining opcodes in the same style:
*        all loads/stores & compares → FIELD(5:5);
*        address-using ops (e.g., Jxx, I/O) → MEMORY with proper bound;
*        ops with no address effect (e.g., NOP/HLT/SRC) → GOOD.)
* -----------------------------------------------------------------------------

        END   BEGIN
```

**How it works**

1. **Front-end** rejects illegal `I` (must be ≤6) and `C` (must be <64), then checks `F ≤ Fmax(C)` from the table.
2. **Dispatch** jumps to an **opcode-specific checker** from the same table entry:

   * `FLOAT` accepts the special case `F=6` for arithmetic; otherwise it falls through to `FIELD`.
   * `FIELD` verifies a partial field by dividing `F` by 9 (the classic MIX trick), accepting iff `0 ≤ L ≤ R ≤ 5`.
   * `MEMORY` ensures that when `I=0` the address (or I/O unit) is a valid nonnegative value within bounds (3999 for memory, or a tighter bound for I/O/MOVE supplied via the table).
3. If all checks pass, control reaches **GOOD**; otherwise it reaches **BAD**.

This is exactly the “table + dispatch” approach Knuth recommends: it’s short, fast, and easy to extend—just adjust the two numbers in each table entry if an opcode’s rules change.

### 10 [31]

Here’s a compact, fully annotated MIXAL solution for the **9×8 saddle point** problem.
Matrix element $a_{ij}$ lives at address $1000+8i+j$ for $1\le i\le 9,\;1\le j\le 8$.
The program scans each row, picks the **row minimum** $a_{i j^{*}}$, then checks whether it is also the **column maximum** in column $j^{*}$. If a saddle point is found, it leaves its **address** in `rI1` and halts; otherwise it halts with `rI1 ← 0`.

```mixal
* TAOCP v1 §1.3.2 — Ex.10  Saddle point in 9x8 matrix
* a(i,j) is stored at ADDR = 1000 + 8*i + j  (1-based i,j)

        ORIG 3000

ROWS    EQU  9
COLS    EQU  8
ROW1    EQU  1009          * addr of a(1,1)  = 1000 + 8*1 + 1
COLBAS  EQU  1008          * addr of a(1,0)  so COLBAS + j = addr of a(1,j)

* Scratch
MINVAL  CON  0             * current row minimum value
MINJ    CON  0             * column index j* (1..8) of that minimum
RSTART  CON  0             * address of a(i,1) for current row
TMP     CON  0             * temp cell for building rI1 result

* rI1: result (address or 0)
* rI3: remaining rows counter
* rI4: inner loop counter
* rI5: row pointer (address of current element in row)
* rI6: column pointer for column check

START   ENT1 0             * rI1 ← 0 by default (means "no saddle")
        ENT3 ROWS          * rI3 ← 9 rows to process
        ENT5 ROW1          * rI5 ← address of a(1,1)

RLOOP   ST5  RSTART        * remember row start address a(i,1)
        ENT2 1             * rI2 ← j = 1
        LDA  0,5           * A ← a(i,1)
        STA  MINVAL        * row minimum so far
        ST2  MINJ          * j* ← 1
        ENT4 COLS-1        * rI4 ← 7 more elements to scan in this row

* ---- scan the rest of the row, keep the minimum and its column j* ----
RNEXT   INC5 1             * move to a(i,j+1)
        INC2 1             * j ← j+1
        LDA  0,5           * A ← a(i,j)
        CMPA MINVAL
        JGE  RKEEP         * if a(i,j) >= current min, keep old min
        STA  MINVAL        * else new minimum found
        ST2  MINJ
RKEEP   DEC4 1
        J4P  RNEXT

* ---- check whether the row-minimum is also a column-maximum ----
        LD6  MINJ          * rI6 ← j*
        INC6 COLBAS        * rI6 ← 1008 + j*  (points at a(1,j*))
        ENT4 ROWS          * rI4 ← 9 rows to test

CCHECK  LDA  0,6           * A ← a(k,j*)
        CMPA MINVAL
        JG   NOTSAD        * some a(k,j*) > MINVAL ⇒ not a saddle
        INC6 8             * next row: address +8
        DEC4 1
        J4P  CCHECK

* Success: build the exact address 1000+8*i+j* = RSTART + (j*−1), put in rI1
        LDA  RSTART
        ADD  MINJ
        SUB  =1=
        STA  TMP
        LD1  TMP           * rI1 ← candidate address
        HLT                * done (saddle point found)

* ---- move to next row and continue the search ----
NOTSAD  LD5  RSTART        * restore this row's start address
        INC5 8             * advance to next row's a(i+1,1)
        DEC3 1
        J3P  RLOOP

        ENT1 0             * no saddle point anywhere
        HLT

        END  START
```

Why it works. For each row $i$ the first loop walks from `a(i,1)` to `a(i,8)` keeping the smallest value and its column $j^{*}$. The second loop fixes the column $j^{*}$ (pointer `rI6 = 1008 + j^{*}`) and marches down by 8 through all nine rows; if any entry exceeds the row minimum, the candidate cannot be a saddle point. Passing that test means

$$
a_{i j^{*}} \le a_{i k}\;\forall k\quad\text{and}\quad
a_{i j^{*}} \ge a_{k j^{*}}\;\forall k,
$$

so $a_{i j^{*}}$ is a saddle point and we return its **memory address** in `rI1`. If none is found after all 9 rows, `rI1` remains 0.

Here $m=9$ rows and $n=8$ columns.

(a) All 72 entries are distinct (all $72!$ orders equally likely)

For a fixed cell $(i,j)$ look only at the $m+n-1$ entries in its row or column:
$\{a_{i1},\dots,a_{in}\}\cup\{a_{1j},\dots,a_{mj}\}$ (with $a_{ij}$ counted once).

To be a saddle, $a_{ij}$ must be **smaller** than the other $n-1$ in its row and **larger** than the other $m-1$ in its column. In the total order of these $m+n-1$ items this means exactly:

* the $m-1$ smallest are the other column-$j$ entries (in any order),
* then comes $a_{ij}$,
* then the $n-1$ remaining row-$i$ entries (in any order).

The number of favorable orders is $(m-1)!(n-1)!$; all $(m+n-1)!$ orders are equally likely. Hence

$$
\Pr\{(i,j)\text{ is a saddle}\}=\frac{(m-1)!(n-1)!}{(m+n-1)!}.
$$

With all entries distinct there can be **at most one** saddle point (two would force a strict inequality cycle), so the probability the matrix has a saddle is just $mn$ times the above:

$$
\boxed{\Pr\{\text{saddle exists}\}
= \frac{mn\,(m-1)!(n-1)!}{(m+n-1)!}}.
$$

For $m=9,\;n=8$,

$$
\Pr = \frac{72\cdot 8!\cdot 7!}{16!}
= \frac{1}{1430}
\approx 6.993\times 10^{-4}.
$$


(b) Entries are i.i.d. 0/1 with all $2^{72}$ matrices equally likely

A cell is a saddle iff either
• it is a **0** and its **column is all zeros**, or
• it is a **1** and its **row is all ones**.

Thus a saddle exists iff there is **at least one all-zero column** or **at least one all-one row**. These two events are disjoint (they cannot happen together because their intersection cell would have to be both 0 and 1).

Columns are independent, so

$$
\Pr\{\text{some column all 0}\}
=1-\bigl(1-2^{-9}\bigr)^8.
$$

Rows are independent, so

$$
\Pr\{\text{some row all 1}\}
=1-\bigl(1-2^{-8}\bigr)^9.
$$

Therefore

$$
\boxed{\Pr\{\text{saddle exists}\}
= \bigl[1-(1-2^{-9})^{8}\bigr]
+ \bigl[1-(1-2^{-8})^{9}\bigr]}
= \frac{924744796234036231}{2^{64}}
\approx 0.0501305159.
$$

So: about $0.0699\%$ in the distinct-entries case, and about $5.01\%$ in the 0/1 case.

### 12 [HM42]
**The algorithms**

- **Sol. 1 (row-min then check its column):** For each row, scan the 8 entries to get the row minimum (and all columns where it occurs), then for each such column scan downward until you see a larger value; if none, you’ve found a saddle.
- **Sol. 2 (min of column-maxima):** Phase 1: compute every column maximum $C(j)$ and then $c_{\min}=\min_j C(j)$. Phase 2: run across each row; if you ever see an entry $<c_{\min}$ the row can’t contain a saddle; otherwise look for an entry $=c_{\min}$ (then you have a saddle).
- **Sol. 3 (suggested):** Same as Sol. 2, but in Phase 1 also remember the row(s) where the minimal column-maximum occurs; search **only** those row(s) in Phase 2 (usually one row when there are no ties).


**Model A: all 72 entries distinct (all orders equally likely)**

Some useful facts for i.i.d. continuous data:

- A row minimum among 8 is typically at the $\approx 1/9$ quantile.
  When you check its column you meet a larger entry with probability $\approx 1-1/9=8/9$ at each step, so the expected number of comparisons in that column is $\approx 1/(8/9)=1.125$.
- To form each column maximum needs 8 comparisons; finding the minimum of the 8 maxima needs 7 more.
  The min of the 8 column-maxima has mean $\mathbb{E}[c_{\min}] \approx 0.7455$ (for uniform\[0,1]). Thus, while scanning a row in Phase 2 of Sol. 2, you hit an entry $<c_{\min}$ after about $1/\mathbb{E}[c_{\min}]\approx 1.34$ comparisons on average.

Rough comparison in **comparisons**:

- **Sol. 1:** $9$ rows × (row-min $=7$ comps + column-check $\approx 1.125$)
  $\Rightarrow \; 9\times(7+1.125)\approx \mathbf{73}$.
- **Sol. 2:** Phase 1 $=8\cdot 8+7= \mathbf{71}$; Phase 2 $=9\times 1.34\approx 12$.
  Total $\approx \mathbf{83}$.
- **Sol. 3:** Phase 1 as in Sol. 2 ($\mathbf{71}$); Phase 2 usually just **one** row scan ($\approx 8$ comps).
  Total $\approx \mathbf{79}$.

**Verdict (distinct case):** **Sol. 1** is best; **Sol. 3** is close (usually one row in Phase 2), **Sol. 2** is a bit slower.


**Model B: entries are i.i.d. 0/1 with all $2^{72}$ matrices equally likely**

Facts from Ex. 11(b):
$\Pr(\text{some all-zero column}) = 1-(1-2^{-9})^{8}\approx 0.0155$,
$\Pr(\text{some all-one row}) = 1-(1-2^{-8})^{9}\approx 0.0346$,
overall saddle probability $\approx 0.0501$.

Expected costs:

- **Sol. 1:** You must scan **every row completely** to build the list of minima (7 comps/row = 63 total).
  In a typical row the number of zeros is $\mathbb{E}=4$; each column test sees a 1 after about 2 comparisons on average, so ≈$4\times 2=8$ comps/row for column checks.
  Total $\approx 9\times(7+8)=\mathbf{135}$ comparisons (much less if you hit an all-ones row early, much more favorable than all-zeros case, but in expectation this ≈135).
- **Sol. 2:** Phase 1 still $\mathbf{71}$ comps.
  Phase 2: when no all-zero column (prob ≈0.9845), $c_{\min}=1$ and each row is ruled out as soon as you see the first 0 (≈2 comps/row, ≈18 total), unless you encounter an all-ones row (prob ≈0.0346), in which case you spend ≈8 comps in that row and stop.
  Expected Phase 2 cost ≈ $0.9654\cdot18+0.0346\cdot(4\!\times\!2+8)\approx 17.9$.
  Total $\approx \mathbf{89}$ comparisons.
- **Sol. 3:** Offers **no advantage** here: ties in column-maxima are ubiquitous (usually all columns have max 1, or if a zero column exists it tags **every** row). So it behaves like Sol. 2 but with extra bookkeeping ⇒ slightly worse than $\mathbf{89}$.

**Verdict (0/1 case):** **Sol. 2** is decisively better on average. (In the extreme all-zero matrix, Sol. 1 wins, but that case has probability $2^{-72}$.)


- **Distinct entries:** pick **Solution 1** (row-min then column-check).
- **0/1 entries:** pick **Solution 2** (min of column-maxima, then short row scans).
- The **suggested Solution 3** (record candidate rows during Phase 1) helps mainly in the distinct-entries setting with few ties; it beats Sol. 2 there but is still typically a hair slower than Sol. 1 for $9\times 8$.

**How it works**

- **Double-buffered I/O.** `BUF0` and `BUF1` are 100-word blocks. We start the next `IN` into the “NEXT” buffer (`rI4`) while we are scanning the “CURRENT” buffer (`rI3`). The `JBUS` after a block has been scanned ensures the next read has completed before we swap the roles. The instruction `SWAB` is the short way to swap `rI3` and `rI4` (if your MIX simulator lacks it, do the swap with `ENTA 0; ENTA 0,3; ENT3 0,4; ENT4 0,3`).

- **First asterisk ends the count.** We hold the “code of `*`” in memory (`AST`), fetched portably from an `ALF` word. When a `*` is seen, we set `FLAG=1` and abruptly end the scan of the current block; the extra block on the tape (as the problem promises) allows the last queued `IN` to finish cleanly.

- **Letter test.** The three `ALF` helpers give codes for `*`, space, and `A`. For a byte `c`, we check
  `c != blank`, `A ≤ c ≤ Z`. If so we index the `CNT[1..26]` table by `j = c − 'A' + 1` and increment.

- **Printing.** We reuse the line-setup trick from Program **P**:
  `CHAR` converts a number to 10 decimal characters in `A‖X`. We then store the six rightmost digits into the beginning of the 24-word `LINE` buffer (to the right of the "letter + space"), and `OUT` one printer line (24 words). Letters with zero counts are skipped.

> How often do the Gregorian Easter dates repeat?

Answer: the full sequence repeats every **5,700,000 years**.

Why:

- Easter depends on the 19-year Metonic (lunar) cycle, the 400-year Gregorian solar cycle, a 25-year correction cycle used in the epacts, and the 30 possible epact values.

For a fixed $n$, write

We need

$$

So we can add $(m_h-m+1)\,t$ all at once and jump to the next block.

Yes—the formula is **true**:

$$
\boxed{\displaystyle \lim_{n\to\infty}\big(S_{n+1}-S_n\big)=\ln 10 } .
$$

**Proof sketch**

Let $r_n(x)=\frac{\lfloor 10^{n}x+\tfrac12\rfloor}{10^{n}}$ and

$$
S_n=\sum_{m\ge1} r_n\!\left(\frac1m\right).
$$

Since $r_n(1/m)=0$ for $m>2\cdot 10^n$, put $A:=10^n$ and write

$$
S_n=\frac1A\sum_{m\le 2A}\Big\lfloor \frac{A}{m}+\tfrac12\Big\rfloor
=\frac{1}{A}\,F(A).
$$

Split

$$
F(A)=\sum_{m\le 2A}\Big\lfloor \frac{A}{m}\Big\rfloor
+\sum_{m\le 2A}\mathbf 1_{\{\{A/m\}\ge \tfrac12\}}
=:H(A)+T(A),
$$

where $\{x\}$ is the fractional part and $\mathbf 1$ an indicator.

1. A classical estimate (Dirichlet’s divisor summatory formula) gives

$$
H(A)=A\log A+(2\gamma-1)A+O(\sqrt A)
$$

($\gamma$ is Euler’s constant).

2. For the second term, observe that $\{A/m\}\ge \tfrac12$ iff there exists an
   integer $k\ge0$ with
   $k+\tfrac12\le A/m<k+1$,
   equivalently
   $A/(k+1)< m\le A/(k+\tfrac12).$
   Hence

$$
T(A)=\sum_{k\ge0}\Big(\Big\lfloor\frac{A}{k+\tfrac12}\Big\rfloor
-\Big\lfloor\frac{A}{k+1}\Big\rfloor\Big)
= A\!\!\sum_{k\ge0}\!\!\Big(\frac{1}{k+\tfrac12}-\frac{1}{k+1}\Big)+O(\sqrt A)
= A\log 2+O(\sqrt A).
$$

Therefore

$$
F(A)=A\log(2A)+(2\gamma-1)A+O(\sqrt A),
\qquad
S_n=\log(2A)+(2\gamma-1)+O(A^{-1/2}).
$$

With $A=10^n$,

$$
S_{n+1}-S_n
=\log\!\big(2\cdot 10^{n+1}\big)-\log\!\big(2\cdot 10^{n}\big)
+O\!\big(10^{-n/2}\big)
=\log 10+o(1),
$$

so the limit is $\log 10$.

### 19 [M30]
**(a)  $x_{k+1}y_k-x_k y_{k+1}=1$ for all $k$**

Let

$$
t_k=\Big\lfloor\frac{y_k+n}{y_{k+1}}\Big\rfloor,\qquad
x_{k+2}=t_kx_{k+1}-x_k,\quad y_{k+2}=t_k y_{k+1}-y_k,
$$

with $x_0=0,y_0=1$ and $x_1=1,y_1=n$.

Set $\Delta_k=x_{k+1}y_k-x_k y_{k+1}$.
For $k=0$: $\Delta_0=1\cdot1-0\cdot n=1$.

Using the recurrence,

$$
\begin{aligned}
\Delta_{k+1}
&=x_{k+2}y_{k+1}-x_{k+1}y_{k+2}\\
&=(t_kx_{k+1}-x_k)y_{k+1}-x_{k+1}(t_k y_{k+1}-y_k)\\
&=x_{k+1}y_k-x_k y_{k+1}=\Delta_k .
\end{aligned}
$$

Hence $\Delta_k$ is constant, and since $\Delta_0=1$ we have

$$
\boxed{x_{k+1}y_k-x_k y_{k+1}=1\quad\text{for all }k\ge0.}
$$

Consequences: $\gcd(x_k,y_k)=1$ (any common divisor would divide 1), so all
fractions $x_k/y_k$ are reduced.


**(b)  The fractions $x_k/y_k$ are the Farey series of order $n$**

From the definition of $t_k$,

$$
t_k y_{k+1}\le y_k+n<(t_k+1)y_{k+1}.
$$

Subtract $y_k$ to get

$$
y_{k+2}=t_k y_{k+1}-y_k\le n
\quad\text{and}\quad
n<y_{k+2}+y_{k+1}.
$$

Thus every denominator satisfies $y_k\le n$, and successive denominators satisfy $y_k+y_{k+1}>n$.

By part (a) the consecutive fractions satisfy

$$
x_{k+1}y_k-x_k y_{k+1}=1.
$$

This identity implies (i) each is reduced, and (ii) any fraction strictly between $x_k/y_k$ and $x_{k+1}/y_{k+1}$ has denominator at least $y_k+y_{k+1}$.
Since $y_k+y_{k+1}>n$, no fraction with denominator $\le n$ lies between them. Moreover one checks

$$
\frac{x_k}{y_k}<\frac{x_{k+2}}{y_{k+2}}<\frac{x_{k+1}}{y_{k+1}}
$$

(using $x_{k+1}y_k-x_k y_{k+1}=1>0$), so the sequence is strictly increasing.

We start at $x_0/y_0=0/1$ and end when $(x_{k+1},y_{k+1})=(1,1)$, hence the whole list runs

$$
0/1,\;\dots,\;1/1,
$$

with all terms reduced, denominators $\le n$, in increasing order, and no additional reduced fraction of order $\le n$ between neighbors. Therefore it is exactly the Farey series of order $n$.

        LDA  ALLOW

        JOV  SETR           * Jumps iff switch has been pressed; clears OV
        JMP  RETU

**How it works**

- Default is Del Mar green + WALK, Berkeley red + DON’T WALK.
- The loop labelled `DMLP` keeps Del Mar green and samples the overflow every ½ s.
  A request is armed only when Del Mar is green or amber, or when Berkeley is amber (i.e., whenever Berkeley is not green).  This is managed by the `ALLOW` flag.
- Once (i) a request is pending and (ii) Del Mar has been green for at least 30 s, we launch the 12-s pre-amber (8 s flash + 4 s steady DON’T WALK), then 8 s amber, then give Berkeley 20 s green (with its own flash/steady sequence), 5 s amber, and return to Del Mar green.
- During Berkeley green we purposely ignore new presses; people cross on that cycle. During other phases we queue a press (by setting `REQP`), guaranteeing that the next Del Mar green will terminate at the earliest lawful moment (≥30 s with a full 12-s pre-amber).

> Calibrate `HS` once for your MIX timing so that the loop under label `DELAY/H1` takes exactly half a second. After that, all required durations are exact by construction.

TEN     CON   10

RET     CON   0
ONES    CON   0
TENS    CON   0

        ENT1  1

        ADD   =1=
        CMPA  =N+1=               * wrap next
        JL    OKN
        ENTA  1

        JG    OKP
        ENTA  =N=

        INC1  1
        CMP1  =N+1=

        ENT1  1                   * CUR ← 1
        STA   CUR
        ENTA  1;  STA  T          * T ← 1  (first kill)

        ENT2  M
        DEC2  1                   * rI2 ← m-1
ADVLP   LD3   CUR
        LDA   NEXT,3
        STA   CUR
        DEC2  1
        J2P   ADVLP

        LD3   CUR
        LDA   T
        STA   ORD,3

        LDA   PREV,3

        STA   CUR                 * CUR ← next (the one after the dead guy)

        LD4   TMP                 * write NEXT[prev] ← next
        STA   NEXT,4

        LDA   TMP
        STA   PREV,3

        LDA   T
        CMPA  =N=
        JE    PRINT

        ADD   =1=
        STA   T
        JMP   KILL

        ENT1  1
ROW     LD3   ORD,1               * value = ORD[i]
        LDA   0,3
        ENTX  0
        DIV   TEN
        STX   ONES
        STA   TENS

        LDA   ONES;  ADD D0;      STA 0,4(2:2)   * ones
        LDA   SPC;                STA 0,4(3:3)   * space
        STA   0,4(4:4);           STA 0,4(5:5)   * padding blanks

        INC1  1
        CMP1  =N+1=
        JL    ROW

        HLT

```

        LDA   SPC; STA 0,4(1:1); STA 0,4(2:2); STA 0,4(3:3); STA 0,4(4:4)
        LDA   PLUS; STA 0,4(5:5)
        JMP   NEXTC
W3      * bottom row: "+++++"
        LDA   PLUS
        STA   0,4(1:1); STA 0,4(2:2); STA 0,4(3:3); STA 0,4(4:4); STA 0,4(5:5)
        JMP   NEXTC

* ----- outside (-1) square: suppress borders that abut -1 at right/below
OUTM1   LDA   T
        CMPA  =3=
        JE    M1BOT
        * top/mid subrows
        * if right neighbor is -1 → five blanks; else "    +"
        ENTA  B
        ADD   I
        MUL   =25=
        ADD   J
        INC   A ONE
        LDA   0,A
        CMPA  NEG1
        JE    ALLSP
        LDA   SPC; STA 0,4(1:1); STA 0,4(2:2); STA 0,4(3:3); STA 0,4(4:4)
        LDA   PLUS; STA 0,4(5:5)
        JMP   NEXTC
ALLSP   LDA   SPC
        STA   0,4(1:1); STA 0,4(2:2); STA 0,4(3:3); STA 0,4(4:4); STA 0,4(5:5)
        JMP   NEXTC
M1BOT   * bottom subrow
        ENTA  B
        ADD   I
        INC   A ONE
        MUL   =25=
        ADD   J
        LDA   0,A
        CMPA  NEG1
        JE    ALLSP
        LDA   PLUS
        STA   0,4(1:1); STA 0,4(2:2); STA 0,4(3:3); STA 0,4(4:4); STA 0,4(5:5)

NEXTC   INC4 1
        INC2 1
        LDA  J
        CMPA N
        JLE  COL

        OUT  LINE(PRN)

        LDA  T
        ADD  ONE
        STA  T
        CMPA THREE
        JLE  SUBROW

        INC1 1
        LDA  I
        CMPA N
        JLE  ROW

        HLT
        END  START
```


**Notes, hints, and adjustments**

- **Card input**: the code expects 23 cards; each of the first 23 bytes is `'0'` or `'1'`. If your MIX monitor uses a different input device, just change `CARD=16` and the `IN`/`JBUS` calls accordingly.
- **Edge removal** (the “−1 flood”): we iterate until a full pass makes no change; for a 23×23 matrix this is trivial in time.
- **Numbering rule**: a white square gets a number if it begins a **down** word (white below, not white above) or an **across** word (white right, not white left). Numbers are two digits with a **leading zero** as in the book’s figure.
- **Exact line lengths**: each printer line has 23 boxes × 5 bytes = 115 bytes; the line buffer has 24 words (120 bytes), so the last five are blank and harmless.
- You can change `B`, `NUM`, or `LINE` bases if they clash with your monitor; all addresses are absolute here to keep the program self-contained.
