# 2.3 Trees

Notation:

- A rooted tree has one distinguished root.
- Degree of a node = number of children.
- A terminal node (leaf) has degree 0.
- In Dewey notation, node `a1.a2....ak` means: take child `a1` of root, then child `a2`, etc.

### 1 [06]
How many different rooted ordered trees can be formed with node labels `A,B,C`?

Count by root choice and shape.

- Root choice: 3 ways.
- For each root, either:
  - two children at same level (2! arrangements), or
  - one child which itself has one child (2! arrangements).

So per root: `2 + 2 = 4`, total `3*4 = 12`.

**Answer:** 12.

### 2 [06]
How many oriented (unordered sibling) trees correspond in the same setting?

When sibling order is ignored, the first pairings collapse.

- Root choice: 3.
- For each root: two structural types (star vs chain).

Total `3*2 = 6`.

### 3 [08]
Prove each node has a unique path to the root.

In a rooted tree, every nonroot node has exactly one parent. Repeatedly following “parent” must terminate at root (acyclic finite ancestor relation). Because each step has exactly one choice, the sequence is unique.

### 4 [06]
Do higher levels appear further down the page in top-to-bottom drawings?

Yes, by drawing convention level number increases downward. Geometric placement encodes depth.

### 5 [06]
If `A` has three siblings and parent `B`, what is degree of `B`?

If `A` has three siblings, parent `B` has four children total.

**Answer:** degree(`B`) = 4.

### 6 [10]
Interpret “`X` and `Y` are `m`th cousins, `n` times removed” in tree language.

They have a nearest common ancestor such that each is descended from distinct children lines of that ancestor; cousin number controls ancestor depth offset, and “removed” is the difference in depths of `X` and `Y` from that common ancestor.

### 7 [10]
Generalize kinship relation in tree terms.

All genealogical terms reduce to:

1. choose least common ancestor (LCA),
2. measure depths `dx, dy` from LCA,
3. derive relation from `(dx,dy)`.

So classification is a function of LCA and depth pair.

### 10 [10]
Do nested nonempty sets always represent a tree?

Not always. A forest or structures lacking a unique global root may satisfy local nesting/disjointness but fail connected single-root requirement. Hence extra root-connectedness condition is needed.

### 11 [M20]
Infinite tree axioms: what changes?

Most finite-tree parent/ancestor definitions extend if we require:

- unique root,
- unique parent for nonroot nodes,
- finite unique path from each node to root.

Cardinality may be infinite; local axioms remain.

### 12 [12]
When does a partially ordered set correspond to a tree/forest?

For rooted tree order (`x<=y` meaning ancestor):

- each principal ideal is a chain,
- every nonminimal element has unique immediate predecessor,
- minimal element(s) are roots (one for tree, many for forest).

### 13 [10]
If node has Dewey number `a1.a2....ak`, what numbers occur on path to root?

Successive prefixes:

`a1.a2....ak`, `a1.a2....a{k-1}`, ..., `a1`.

(And then root marker.)

### 14 [16]
Show finite set `S` of Dewey numbers defines a tree when closure condition holds.

Condition: if `alpha.m in S`, then `alpha.(m-1) in S` for `m>1`, and `alpha in S` for `m=1`.

This guarantees every node’s parent prefix exists; finite closure yields unique rooted prefix-closed structure. Hence valid rooted ordered tree.

### 15 [10]
Invent Dewey-style notation for binary trees.

Use symbols `{1,2}` per step:

- `1` = left child,
- `2` = right child.

Root is empty code (or `0`). Example `2.1.1` = right-left-left descendant.

### 16 [12]
Draw expression trees for `2(a-b/c)` and `a+b+5c`.

- `2(a-b/c)`: root `*`, left child `2`, right child `-`, with `/` under right subtree.
- `a+b+5c`: root `+`, one child `a`, another `b`, and subtree `*` with `5,c` (depending binary association).

### 20 [20]
0-2 trees: show odd node count and bijection with binary trees on `n` nodes.

If internal nodes have exactly 2 children, let internal count `i`, leaves `l`.
Edges = `2i`; also edges = `i+l-1`; hence `l=i+1`. Total nodes `N=i+l=2i+1` odd.

Bijection with binary trees:

- replace each null child pointer of binary tree by a leaf node;
- each original node becomes internal degree-2 node.

Binary tree with `n` nodes has `n+1` null pointers, so resulting 0-2 tree has `n+(n+1)=2n+1` nodes. Reversible by deleting added leaves.

### 21 [14]
If a tree has `n1,n2,...,nm` nodes of degrees `1,2,...,m`, how many leaves?

Let leaves be `n0`. Total nodes `N = n0 + n1 + ... + nm`.
Sum of degrees equals edges: `n1 + 2n2 + ... + m nm = N-1`.
Substitute `N` and solve:

\[
n0 = 1 + \sum_{k=1}^{m}(k-1)n_k.
\]

### 22 [12]
Give binary-tree representation idea for European paper sizes.

Use root `A0`; each split produces two children of next size (`A1`), and recursively each `Ak` node splits into two `A{k+1}` nodes. This forms a full binary refinement tree encoding paper halving hierarchy.
