# 1.1 算法

### 1 [10]
最小就地算法（5 个替换）```
t ← a
a ← b
b ← c
c ← d
d ← t
```
演练。
设初始值为“a0，b0，c0，d0”。

1. `t ← a` → `t = a0`
2. `a ← b` → `a = b0`
3. `b ← c` → `b = c0`
4. `c ← d` → `c = d0`
5. `d ← t` → `d = a0`

最终状态是“(a, b, c, d) = (b0, c0, d0, a0)”，正是所请求的左旋转。

为什么 5 次更换是最佳选择

1. 所有四个变量都发生变化。在 4 周期“(a b c d)”中，“a、b、c、d”均不保留其原始值，因此这四个周期中的每一个都必须至少分配一次 → ≥ 4 次替换。

2. 覆盖时必须保留一个原始值。
   一旦你赋值，比如“a ← b”，原始的“a0”就会丢失，除非保存它。但最后的“d”必须变成“a0”。因此，至少需要一次额外的“保存”（例如，`t ← a`）→ ≥ 4 + 1 = 5 次替换。

因此 5 是一个下界，上面的过程实现了它，所以它是最小的。

概括

对于 n 循环旋转 `(x1 x2 … xn)`，相同的参数显示 n + 1 次替换的下界（n 设置每个变量一次，加上 1 保存值），以及标准方案```
t ← x1;  x1 ← x2;  x2 ← x3;  …;  xn−1 ← xn;  xn ← t
```
满足该界限。

### 2 [15]
考虑一下我们在完成之前通过 E1/E2/E3 后返回 E1 的任何时候。

* 在上一次通过 E1 时，我们计算了余数 $r$，其中 $0\le r<n$。如果 $r=0$ 我们就会在 E2 处停止并且不会再次到达 E1。因此，每当我们*确实*再次到达 E1 时，我们必须有 $r\neq 0$，即 $0<r<n$。
* 然后E3设置$m\leftarrow n$和$n\leftarrow r$。因此，在下一个E1开始时，我们有  $$
  m_{\text{new}} = n_{\text{old}} \quad\text{and}\quad n_{\text{new}} = r_{\text{old}}\,,
  $$
与 $0<r_{\text{old}}<n_{\text{old}}$。因此$m_{\text{new}} = n_{\text{old}} > r_{\text{old}} = n_{\text{new}}$，即严格来说$m>n$。

这表明从第二次访问 E1 开始，严格的不等式 $m>n$ 始终成立。

对于第一次访问 E1，不存在这样的保证：输入可能有 $m\le n$ （实际上允许 $m<n$），因此该语句允许异常“可能第一次除外”。 （如果在文本中建议添加可选的“ensure $m\ge n$”交换步骤，则该属性也将在第一次保留。）

### 3 [20]
算法 F（Euclid，无“琐碎替换”）

目标。给定正整数“m”和“n”，计算“gcd(m, n)”。

- F1。 \[余数 m / n] 将 `m` 除以 `n` 并将余数替换为 `m`。 （所以`0 ≤ m < n`。）
- F2。 \[它为零吗？] 如果“m = 0”，则停止并回答“n”。
- F3。 \[余数 n / m] 将 `n` 除以 `m` 并将余数替换为 `n`。 （所以`0 ≤ n < m`。）
- F4。 \[它为零吗？] 如果“n = 0”，则停止并回答“m”；否则转F1。

这个版本从不执行“`m ← n, n ← r`”；我们每次都直接用余数覆盖股息并交替角色。 

为什么这是正确的

1) 不变式（保留gcd）

每一步，
`gcd(m, n) = gcd(n, m mod n)` 和 `gcd(m, n) = gcd(m, n mod m)`。
这是欧几里得方法背后的基本属性； F1/F3 正是这些替换，因此 gcd 永远不会改变。 （这与证明算法 E 的推理相同；请参阅第 1.1 节。([PagePlace][1])）

2) 终止

每个余数都严格小于其除数。 F1 之后我们有 `m < n`； F3 之后我们有 `n < m`。因此，每次通过 F1-F4 时，“{m, n}”对的最大值都会严格减小。由于最大值是非负整数，因此它只能减少有限多次，因此算法停止。

3）停下来后，答案是正确的

如果我们停在 F2 处，则 `m = 0`，因此 `gcd(m, n) = gcd(0, n) = n`。
如果我们停在 F4 处，则 `n = 0`，因此 `gcd(m, n) = m`。
与不变量相结合，这等于原始的“gcd”。


工作示例

使用 F 计算 `gcd(2166, 6099)`。

- F1：`m ← 2166 mod 6099 = 2166`（因为 2166 < 6099）
  F2: `m ≠ 0` → 继续
- F3: `n ← 6099 mod 2166 = 1767`
  F4: `n ≠ 0` → 循环
- F1: `m ← 2166 mod 1767 = 399`
  F2：`m ≠ 0`
- F3: `n ← 1767 mod 399 = 171`
  F4：`n≠0`
- F1: `m ← 399 mod 171 = 57`
  F2：`m ≠ 0`
- F3: `n ← 171 mod 57 = 0`
  F4: `n = 0` → 停止并给出答案 `m = 57`。
  
  
### 4 [16]
我们反复划分并保留余数； gcd 是最后一个非零余数。1. $6099 = 2\cdot 2166 + 1767$ → $r_1=1767$
2. $2166 = 1\cdot 1767 + 399$ → $r_2=399$
3. $1767 = 4\cdot 399 + 171$ → $r_3=171$
4. $399 = 2\cdot 171 + 57$ → $r_4=57$
5. $171 = 3\cdot 57 + 0$

最后一个非零余数是 $57$，所以$$
\gcd(2166,6099)=57.
$$
快速检查

- 可整除性：$2166/57=38$ 和 $6099/57=107$ 是整数。
- 没有更大的公因数：欧几里得的过程确保了最优性。

### 5 [12]
“阅读这套书的程序”并不是一个真正的算法，因为它至少违反了 Knuth 的五个标准中的三个：

- 有限性：它循环回到步骤 3，因此不需要终止。
- 确定性：诸如“你累了吗？”之类的步骤或“有趣吗？”是主观的，不精确。
- 有效性：诸如“去睡觉”或“让你的朋友购买副本”之类的操作不是基本的可执行步骤。

它在格式上也与算法 E 不同：它使用普通数字（不是 E1、E2…）、散文而不是箭头符号 ($\leftarrow$)、没有括号内的摘要，也没有明确的结束标记。

因此，这个“程序”更像是一个笑话，而不是一个正确的算法。

### 7 [M21]
令 $A(m,n)$ 为欧几里得算法在正整数有序对 $(m,n)$ 上启动时执行步骤 E1（“将 $m$ 除以 $n$；令 $r$ 为余数”）的次数。

Knuth 将 $T_n$ 定义为 $A(m,n)$ 的平均值，当 $n$ 固定且允许 $m$ 范围为所有正整数时；因为第一次除法后只有 $m\bmod n$ 的余数很重要，所以平均值取 $m=1,2,\dots,n$ （统一）。他指出，对于较大的 $n$，$T_n$ 是 $\asymp c\ln n$，其中 $c=12\ln 2/\pi^2$。

该练习要求类似地定义 $U_m$，其中 $m$ 固定，$n$“范围涵盖所有正整数”，并将 $U_m$ 与 $T_m$ 联系起来。

我们将证明 $U_m$ 是明确定义的（极限平均值存在并且是有限的）并且$$
\boxed{\,U_m = T_m + 1\,}.
$$
$A(m,n)$ 的两个简单恒等式

(一)当第一个数较小时交换一次。
如果$m<n$，那么欧几里得算法中的第一次除法留下余数$r=m$；下一个状态是$(n,m)$。因此$$
A(m,n) \;=\; 1 + A(n,m)\qquad(m\ne n).
\tag{1}
$$
(II) 当 $n>m$ 时 $n \pmod m$ 的周期性。
将 $n=qm+r$ 写为 $0\le r<m$。 $(m,n)$ 上的第一步给出余数 $r=m$ （因为 $n>m$），所以在一步之后我们就到了 $(n,m)$。再一步将 $n$ 除以 $m$ 并留下余数 $r$；从那时起我们继续$(m,r)$。因此$$
A(m,n) \;=\;
\begin{cases}
2, & r=0,\\
2 + A(m,r), & 1\le r\le m-1.
\end{cases}
\tag{2}
$$
因此，对于 $n>m$，$A(m,n)$ 仅取决于 $n \bmod m$（当 $m\mid n$ 时等于常数 $2$）。

$U_m$ 的存在（和价值）

定义塞萨罗平均值$$
U_m \;:=\; \lim_{N\to\infty}\frac{1}{N}\sum_{n=1}^N A(m,n),
$$
如果限制存在。

将 $1,\dots,N$ 划分为长度为 $m$ 的完整块以及最后一个部分块。使用（II），每个完整的块贡献$$
\sum_{r=0}^{m-1} A\bigl(m,\,qm+r\bigr)
= 2 + \sum_{r=1}^{m-1}\bigl(2 + A(m,r)\bigr)
= 2m + \sum_{r=1}^{m-1} A(m,r).
$$
因此块平均值是$$
\frac{1}{m}\left(2m + \sum_{r=1}^{m-1} A(m,r)\right)
= 2 + \frac{1}{m}\sum_{r=1}^{m-1} A(m,r).
\tag{3}
$$
最后的部分块贡献 $O(m)$，除以 $N$ 后可以忽略不计。因此，定义 $U_m$ 的限制存在并且等于块平均值：$$
U_m \;=\; 2 + \frac{1}{m}\sum_{r=1}^{m-1} A(m,r).
\tag{4}
$$
与$T_m$的关系

根据定义（参见介绍 $T_n$ 的段落以及如何对一组完整的残基求平均值），我们有$$
T_m \;=\; \frac{1}{m} \sum_{x=1}^{m} A(x,m)
= \frac{1}{m}\Bigl(A(m,m) + \sum_{x=1}^{m-1} A(x,m)\Bigr).
$$
现在将 (I) 与 $x<m$ 结合使用：$A(x,m)=1+A(m,x)$。也$A(m,m)=1$。因此$$
T_m
= \frac{1}{m}\Bigl(1 + \sum_{x=1}^{m-1} (1 + A(m,x))\Bigr)
= \frac{1}{m}\Bigl(m + \sum_{x=1}^{m-1} A(m,x)\Bigr)
= 1 + \frac{1}{m}\sum_{x=1}^{m-1} A(m,x).
\tag{5}
$$
比较 (4) 和 (5) 立即得出$$
\boxed{\,U_m = T_m + 1\,}.
$$
对一个小例子进行健全性检查

取$m=5$。可以算出：$$
A(5,1)=1,\;
A(5,2)=2,\;
A(5,3)=3,\;
A(5,4)=3.
$$
然后$$
T_5 = 1 + \tfrac{1}{5}(1+2+3+3)=1+\tfrac{9}{5}= \tfrac{14}{5},
\qquad
U_5 = 2 + \tfrac{1}{5}(1+2+3+3) = 2+\tfrac{9}{5}=\tfrac{19}{5}
= T_5+1.
$$
（手动检查长期运行的 $n$ 值显示区块平均值等于 $19/5$，正如预测的那样。）

后果

由于 $T_n \sim \frac{12\ln 2}{\pi^2}\ln n$ 对于大 $n$，我们得到$$
U_m \sim \frac{12\ln 2}{\pi^2}\ln m + 1,
$$
所以$U_m$ 和$T_m$ 渐近地相差$1$。&#x20;

这证明 $U_m$ 定义良好，并建立了与 $T_m$ 的精确关系。

### 8 [M25]
以 Knuth 的 $(T_j,\;s_j,\;a_j,\;b_j)$ 格式设计一组*字符串重写*规则，从输入字符串 $a^m b^n$ 开始（即 `a` 的 $m$ 个副本，后跟 `b` 的 $n$ 个副本），以字符串 $a^{\gcd(m,n)}$ 停止。这些规则必须以第 1.1 节中使用的表格样式表示为马尔可夫算法。 

快速回顾：$(T_j,s_j,a_j,b_j)$ 的含义

程序状态是一对 $(\sigma, j)$：当前字符串 $\sigma$ 和当前规则索引 $j$。每个表行 $j$ 有：

- $T_j$ (“pattern”)：要搜索的子字符串，
- $s_j$（“替换”）：找到$T_j$时替换的字符串，
- $b_j$：如果确实发生替换，则下一个规则索引，
- $a_j$：如果在$\sigma$中没有找到$T_j$，则下一个规则索引。

操作上（马尔可夫规则）：如果出现 $T_j$，则用 $s_j$ 替换 *leftmost* 出现的位置并跳转到 $b_j$；否则跳转到$a_j$。一个特殊索引 $N$ 表示“停止”。 （[堆栈溢出][1]）

最小工作规则集

使用字母表$\{a,b,c\}$和五个规则（索引$0,1,2,3,4$）；令 $5$ 为停止索引 $N$。

| j | $T_j$ | $s_j$ | $b_j$ | $a_j$ |直觉|
| - | -------- | ------------- | ----:| ----:| ---------------------------------------------------------------- |
| 0 | `ab` | （空）|     1 |     2 |用一个“b”取消一个“a”（一个减法步骤）。              |
| 1 | （空）| `c`（前置）|     0 |     0 |每次取消后，通过在左侧粘贴“c”来计数。 |
| 2 | `a` | `b` |     2 |     3 |将所有剩余的“a”变成“b”。                             |
| 3 | `c` | `a` |     3 |     4 |然后将所有“c”变成“a”。                                  |
| 4 | `b` | `b` |     0 |     5 |如果剩下任何“b”，则重复另一轮欧几里得；否则停止。     |

为什么这有效（高级）：

- 规则0↔1模拟*重复减法*：删除的每个`ab`对应于从较大的数字中减去较小的数字一次；您可以执行此操作的次数为 $\min(m,n)$。我们通过累积“c”来“计算”这些取消。
- 当没有“ab”剩余时，字符串看起来像“c^{min(m,n)} x^{|m-n|}”，其中 $x$ 是幸存的字母（“a”或“b”）。- 规则 2 和 3 将字符串标准化为 `a^{min(m,n)} b^{|m-n|}`，即我们将 $(m,n)$ 对映射到 $(\min(m,n),\,|m-n|)$。
- 如果第二个分量非零，则规则 4 循环；否则停止。这正是基于减法的欧几里得步骤，因此重复的回合以“a^{gcd(m,n)}”结束。上标的意思是“重复‘a’很多次”。 （[堆栈溢出][2]）

正确性论证（简短但完整）

规则 0↔1 循环结束后（规则 2 之前）不变：
令当前对为$(m,n)$。重复删除最左边的“ab”（规则 0），然后在前面加上“c”（规则 1）恰好 $\min(m,n)$ 次，会生成以下形式的字符串：$$
c^{\min(m,n)}\; y^{|m-n|},
$$
其中，如果 $m>n$，则 $y$ 为“a”；如果 $n\ge m$，则为“b”。这编码了“我们从较大的 $\min(m,n)$ 次中减去较小的 $\min(m,n)$ 次。”

标准化（规则2→3）：

- 规则 2 将任何杂散的“a”变成“b”，因此只剩下“c”和“b”。
- 规则 3 将每个“c”变成“a”。
  因此，字符串按照规范顺序变为“a^{\min(m,n)} b^{|m-n|}”。

欧几里德步骤和终止（规则 4）：

- 如果仍有任何“b”，则规则 4 跳回规则 0，因此新对为 $(m',n')=(\min(m,n),|m-n|)$。
- 通过欧几里得算法的减法形式重复变换$(m,n)$，直到$n'=0$；此时规则 4 会下降到 $5$（停止），字符串为“a^{gcd(m,n)}”。 （[堆栈溢出][1]）

工作示例

取$m=6$，$n=10$；以“aaaaaabbbbbbbbbb”开头。

- 第 1 轮（穷尽规则 0）：删除“ab”六次，同时累积“c”→“ccccccbbbb”。
  标准化（规则 2→3）→ 已经没有 `a`，则 `c→a` ⇒ `aaaaaabbbb` = `a^6 b^4`。
- 第 2 轮：取消 `ab` 四次 → `cccc a a` （即 `ccc caa`，但在标准化之前顺序并不重要）⇒ 标准化 ⇒ `a^4 b^2`。
- 第 3 轮：取消两次 ⇒ 标准化 ⇒ `a^2 b^0`。
- 第 4 轮：规则 4 看到“没有 `b` 留下”⇒ 以 `aa` = $a^{\gcd(6,10)}$ 停止。

与欧几里得算法 E 的关系

规则 0→…→4 的每一次完整传递都会实现映射$$
(m,n)\;\mapsto\;(\min(m,n),\;|m-n|),
$$
即，欧几里得算法的除法步骤被*重复减法*取代。这正是高德纳给出的暗示；弦系统只是这个想法的马尔可夫算法呈现。 

注意事项和常见困惑

- “$a^{\gcd(m,n)}$”中的上标表示“`a`重复$\gcd(m,n)$次”。例如，如果 $\gcd(12,18)=6$，则输出为 `aaaaaa`。
- 在规则表中，$b_j$ 是“如果发生替换则分支”，$a_j$ 是“如果没有发生替换则分支”。这种分支方案与最左替换一起是马尔可夫算法的标准，并且符合 Knuth 的 §1.1 形式主义。

### 9 [M30]
让$$
C_1=(Q_1, I_1, \Omega_1, f_1),\qquad
C_2=(Q_2, I_2, \Omega_2, f_2)
$$
是计算方法（状态空间$Q$，初始/允许输入$I\subseteq Q$，停止/“输出”状态$\Omega\subseteq Q$，以及（部分）下一状态函数$f$）。 $C_i$ 的*计算序列*是通常的链 $x, f_i(x), f_i^{(2)}(x),\dots$ 直到（并且如果）达到 $\Omega_i$ 中的状态。

直观上，$C_2$ 可以执行更多、更小的步骤，并且可以在其状态中保留额外的簿记，但它应该忠实地实现 $C_1$。

我们说 $C_2$ 代表 $C_1$ 当且仅当存在总地图

- 编码 $e:Q_1\to Q_2$ 和
- 解码/抽象 $\pi:Q_2\to Q_1$

这样 $\pi\!\circ e=\mathrm{id}_{Q_1}$ （即 $e$ 将 $C_1$ 的抽象状态嵌入到 $C_2$ 的具体状态中，而 $\pi$ 忘记了额外的细节），并且以下内容成立：

1. 输入兼容性。 $e(I_1)\subseteq I_2$。

2.输出兼容性。 $\pi(\Omega_2)\subseteq \Omega_1$ 并且对于每个 $\omega\in\Omega_1$ 都存在一些 $\hat \omega\in\Omega_2$ 和 $\pi(\hat \omega)=\omega$。
   （等效地，$\pi^{-1}(\Omega_1)$ 仅包含 $C_2$ 可以在不改变 $\pi$ 值的情况下到达 $\Omega_2$ 的状态。）

3.步数对应（口吃模拟）。
   对于每个 $q_1\in Q_1\setminus \Omega_1$ 和每个 $q_2\in Q_2$ 且 $\pi(q_2)=q_1$，如果定义了 $f_1(q_1)$ 并且等于 $q_1'$，则存在有限的 $t\ge 1$ 和 $C_2$ 中的路径   $$
   q_2 = r_0 \xrightarrow{f_2} r_1 \xrightarrow{f_2} \cdots \xrightarrow{f_2} r_t
   $$
这样   $$
   \pi(r_j)=q_1\quad(0\le j<t),\qquad \pi(r_t)=q_1'.
   $$
（因此，在实现一个抽象的 $f_1$ 步骤之前，$C_2$ 可能会“卡顿”——采取几个不改变抽象状态的内部步骤。）

4. 终止保存（整个运行的细化）。
   对于每个输入 $x\in I_1$，如果从 $x$ 运行的 $C_1$ 在 $k$ 步中停止并输出 $\omega\in\Omega_1$，则从 $e(x)$ 运行的 $C_2$ 在 $T\ge k$ 步后停止在 $\hat \omega\in\Omega_2$ 状态，其中 $\pi(\hat \omega)=\omega$。

这是细化中使用的标准“带口吃的正向模拟”表征：$C_2$ 可以采取更多（更细粒度）步骤并存储更多信息，但是当您使用 $\pi$ *抽象*其状态时，您会准确地看到 $C_1$ 的抽象行为。

为什么这抓住了意图

- “更多步骤”：条件 3 允许单个 $f_1$ 步骤使用多个 $f_2$ 步骤，同时保持 $\pi$ 不变，直到发生抽象更改。
- “更多信息”：抽象$\pi$可以忘记寄存器、临时变量、程序计数器等； $\pi\!\circ e=\mathrm{id}$ 确保嵌入时不会丢失抽象信息。
- 正确的输入/输出：条件 1-2 将两台机器在开始和结束处对齐。
- 整体计算：条件4确保每个终止抽象计算都以相同的抽象结果具体实现。

在欧几里得示例中看起来如何

在§1.1中，算法E被形式化为$(Q,I,\Omega,f)$系统，其状态为$(m,n,0,1)$、$(m,n,r,2)$、$(n)$等，以及包含步骤E1-E3的下一状态函数。机器程序 $C_2$ 的状态包括内存、寄存器和程序计数器。定义：

- $e$：将摘要$m,n,r,\ldots$放入指定的寄存器/内存中，并将PC设置为相应E步的代码。
- $\pi$：读回那些指定的寄存器并将该代码块的*任何*机器状态映射到相应的抽象状态（忽略临时、标志、暂存空间等）。

$C_2$ 中的长除法例程将执行许多微步，而 $\pi$ 保持在相同的抽象 E1 状态；完成除法后，$\pi$ 跳转到 E2/E3 抽象状态，这正是 (3) 所要求的口吃行为。当$n=0$时，程序分支到返回序列； $\pi$ 将这些返回序列状态映射到 $C_1$ 的停止集中，建立 (2)–(4)。
